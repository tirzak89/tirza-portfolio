System Migration Customer App Courier App VIP Portal

My Role — Product Manager / Technical Product Lead

  • Managed migration roadmap across multiple concurrent platforms

  • Order creation flow
  • Shipment tracking logic
  • Courier operational workflows
  • VIP seller management portal
  • OMS + API ecosystem alignment
Impact
  • Reduced reliance on external vendors — zero remaining dependency on vendor
  • Faster product development cycles and improved feature deployment agility
  • Enabled internal teams to launch new logistics features independently
  • Improved system integration between customer, courier, and OMS platforms
02
Commercial Platform
Custom Pricing & Agent Empowerment Platform
Pricing System Agent Portal Regional Ops
The Problem

Regional logistics agents needed flexibility to compete locally, but pricing was centrally controlled — preventing localised promotions, reducing competitive advantage in regional markets, and limiting shipment acquisition in smaller cities.

Solution

Built a pricing configuration portal within the VIP platform enabling agents to configure shipping fees within their operational coverage and apply localised pricing strategies.

  • Margin protection logic for HQ with validation rules for pricing overrides
  • Inner-agent shipment classification to prevent pricing conflicts
  • Localised promotional tools to attract new sellers per region
Impact
  • Increased shipment acquisition in agent coverage regions
  • Improved agent retention and engagement with the platform
  • Reduced manual pricing adjustments from HQ teams
  • More competitive local pricing strategy without central overhead
03
API Ecosystem
Open API Platform for E-commerce Integrations
Open API Shopee Tokopedia Blibli Webhook
The Problem

Large e-commerce partners needed automated logistics integration, but existing APIs had slow response times, inconsistent payload structures, limited tracking automation, and poor scalability for high order volumes.

New API Suite Developed
  • Create Order API v2 — programmatic bulk shipment creation for enterprise clients
  • Check Shipping Fee API v2 — real-time pricing calculation and logistics cost estimation
  • Tracking Webhook System — automated status push notifications, reducing client polling requests
Impact
  • Enabled large enterprise integrations with Shopee, Tokopedia, and Blibli
  • Reduced manual shipment creation at scale
  • Improved shipment tracking automation and system reliability
  • Scalable API infrastructure built for high order volumes
04
Core Platform
Order Management System (OMS) Enhancements
OMS Master Data Payment Model
The Problem

The logistics platform lacked structured master data for shipment sources and payment models — causing poor data visibility, difficult client filtering, and inefficient operations monitoring across the OMS.

Key Features Built
  • Order Source Master Data System — stores API credentials, marketplace source, order prefix, and waybill prefix per client
  • Payment at Delivery (PAD) — new shipment payment model where recipients pay the shipping fee, stored in order lifecycle with visibility across Search Order, Waybill, and POD
Impact
  • Improved shipment source tracking and financial reconciliation workflows
  • Enabled new payment model adoption across the platform
  • Improved data integrity across the full OMS
05
Analytics
Logistics Intelligence & Operational Dashboards
Dashboards Courier Analytics SLA Monitoring
The Problem

Operations teams lacked real-time visibility into courier productivity and shipment flow — making SLA performance hard to monitor, package bottlenecks difficult to detect, and courier performance tracking severely limited.

Dashboards Designed
  • Courier Performance Dashboard — tracks Proof of Delivery productivity and courier task completion rates
  • Bagging Monitoring — tracks package consolidation and medium-mile transfer efficiency
  • Vehicle Loading Monitoring — tracks vehicle capacity utilisation and logistics flow optimisation
Impact
  • Faster identification of delivery bottlenecks across the network
  • Improved courier productivity monitoring and SLA performance management
  • Data-driven operational decision making for ops leadership
